Muitas vezes queremos acessar um determinado computador remotamente, mas lembramos que o ip não é fixo, pois provavelmente usamos uma conexão que não nos dá o luxo de ser uma conexão dedicada. Como resolver isso? Usando um serviço de DNS dinâmico. Nesta dica veremos o NO-IP. 

Para instalar o cliente for Linux do noIP, os passos são os seguintes: 

1 - Registre sua conta no www.no-ip.com/. 

2 - Instale o no-ip através do apt-get: 

# apt-get install no-ip

3 - O próximo passo é executar noip2-Linux usando a opção "-C -c" (create config), que cria o arquivo de configuração. Você pode indicar onde 
o arquivo será criado, basta indicá-lo no comando. Nesta etapa ele pedirá o login de usuário e o domínio registrado no site: 

# no-ip -C -c /etc/no-ip.conf 
Auto configuration for Linux client of no-ip.com.
Multiple network devices have been detected. 

No caso aqui ele vai pedir que seja selecionada a placa de rede que se conecta à internet, no meu caso é a interface eth1, ou seja, a opção 1: 

Please select the Internet interface from this list.
By typing the number associated with it.
0       eth2
1       eth1 

Após a escolha da interface padrão que o seu host se conecta pela internet ele vai pedir o email com o qual você se cadastrou no site do no-ip e a senha: 

Please enter the login/email string for no-ip.com "email"
Please enter the password for user 'email' "senha" 

Defina qual host ele vai atualizar o IP (se houver mais de um ele mostra todos). 

5 hosts are registered to this account. 

No meu caso, tenho 5 hosts registrados. 

Caso deseje, informe um programa ou script a ser executado a cada atualização. 

4 - Com o arquivo de configuração criado, inicie o no-ip rodando o comando abaixo: 

# /etc/init.d/no-ip restart 

Pronto, ele já está configurado, funcionando e configurado para iniciar no boot.
